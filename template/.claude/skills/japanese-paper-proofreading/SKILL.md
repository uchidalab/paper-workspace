# 日本語論文の表記・校正規則

日本語の学術文書を「表記レベル」で機械的に整える校正スキル。論証の組み立て・段落構成・トピックセンテンスなど**言語非依存の作法は [[paragraph-writing]] / research-paper-writing に任せ**、ここでは**日本語特有の表記**だけを扱う。提出前・推敲時に下のチェックリストで全文を点検する。

## 大原則

- **正誤が決まるものは、正しい方へ直す**（半角カタカナ・全角数字・規格外の句読点など＝誤り。統一の問題ではない）。
- **正誤が決まらない揺れは、下の「確定スタイル」に従う**。`〜`/`～`、和欧間スペースの有無などは「どちらでも正しい」が、本スキルでは**選択を確定済み**とする。揺れたら多数派ではなく**確定スタイルへ寄せる**。
- 機械的に検出できるものは grep で洗い出してから直す（末尾の検出例を参照）。

## 確定スタイル（house style）

正誤が決まらない選択はここで確定する。今後はすべてこの表に従う（個別に決め直さない）。

| 項目 | 確定 |
|---|---|
| 句読点 | 全角 **，．**（`、。` は使わない） |
| 和欧・数式間スペース | **手動スペースを入れない**（pLaTeX の和欧文間空白に任せる。詳細は下節） |
| 範囲のダッシュ | 全角波ダッシュ **`〜`**（`~`/`～` は使わない） |
| 括弧 | 和文は全角 **（）**、欧文・コード中は半角 `()` |
| カタカナ長音 | 語末長音は**付ける**（例: コンピューター、パラメーター）。半角カタカナ禁止 |
| 箇条書き末尾 | **句点を付けない** |
| 文体 | **である調**（です・ます禁止）。**体言止めは使わない**（見出し・図表キャプション・箇条書きを除く） |
| 数字・単位 | 半角／数式。**全角数字・全角単位は使わない**。3桁区切りは**原則付けない**（5桁以上が出たら半角カンマ、年号は区切らない） |
| 式参照 | **`式\eqref{eq:xxx}`**（amsmath） |
| 図・表参照 | **`図\ref{fig:xxx}` / `表\ref{tab:xxx}`** |
| 節・章参照 | **`\ref{sec:xxx}節`**（番号は直書きしない） |
| 文献引用 | **`語~\cite{key}`**（`~`で密着、語の直後・句読点の前） |
| label 接頭辞 | `fig:` `tab:` `eq:` `sec:` |
| 記号 | 数式記号・演算子は数式モード `$...$`。引用符は和文「」・欧文 `` ``...'' `` |

## 校正チェックリスト

| # | 項目 | 規則 | よくある違反 |
|---|---|---|---|
| 1 | 句読点 | 学術文書は全角の **，．** に統一。`、。` と `，．` を混在させない | 「〜と、クラス」「〜取り組む。」が `，．` の文中に混ざる |
| 2 | 全角・半角 | 英字・数字・数式・`%`・コロン `:` は半角。和文の括弧・読点は全角。**コロンや括弧を全角/半角で混ぜない** | `16：00`(全角コロン) と `16:00`(半角) の混在、`(火)` と `（火）` の混在 |
| 3 | 和欧・数式間スペース | **手動の半角スペースは入れない**（pLaTeX の和欧文間空白に任せる）。下「和欧間スペースの方針」を参照 | 「CLIPの」と「CLIP の」の混在 |
| 4 | ダッシュ・波ダッシュ・ハイフン | 範囲は全角波ダッシュ `〜`、マイナスは数式中の `-`/`$-$`、連結語は `-` | 範囲に `~`(半角波) と `〜`(全角波) の混在 |
| 5 | カタカナ | 半角カタカナ禁止。長音表記（コンピュータ/コンピューター等）を全文で統一 | — |
| 6 | 箇条書き | `itemize`/`enumerate` の各項目末に**句点（．）を付けない**で統一 | 一部だけ句点あり |
| 7 | 文体 | **である調に統一**（です・ます調を混在させない）。体言止めの可否も方針化 | である調の中に「です・ます」混入 |
| 8 | 数値・単位 | **全角数字・全角単位は使わない**（数字・`pt`・`%`・`×` は半角/数式）。単位の付け方とスペース、3桁区切りの有無を統一 | 全角数字、`pt` 前のスペース有無の揺れ |
| 9 | 参照表記 | 下「参照表記の方針」に統一。番号は必ず `\ref`/`\eqref` で出し、直書きしない | 「図1」と「Fig.1」、`\ref` だけで類別詞なし |
| 10 | 略語の初出展開 | 略語は初出でスペルアウト（または定義）し、2回目以降は略語で統一 | 略語が定義なく登場、表記が揺れる |
| 11 | 記号 | **数式記号・演算子は LaTeX の数式モードで書く**（下「記号の方針」）。地の文に裸の Unicode 記号を置かない | 裸の `×`・`＞`、`“ ”` の混在 |

## 和欧間スペースの方針（確定: 手動で入れない）

英字・数式の前後に**手動の半角スペースは入れない**で確定。理由:

- pLaTeX/upLaTeX は和文と欧文・インライン数式の境界に**和欧文間空白（xkanjiskip, 既定 1/4 全角）を自動挿入**する。手動スペースは不要。
- 手動スペースは**本質的に非対称**で不統一になる。pLaTeX は**和文直後の半角スペースを無視**するため、「の CLIP」のスペースは消える一方、「CLIP の」のスペースだけが残る。片側しか効かないので必ず揺れる。
- 自動の xkanjiskip に任せれば、両側とも同じ空き量で**自動的に統一**される。空きが狭いと感じるなら本文ではなくプリアンブルで `\setlength{\xkanjiskip}{...}` を調整する。

すでに手動スペースが入っている既存原稿は、校正時に**和文と隣接する半角スペースを削除**して確定スタイルへ揃える。

## 参照表記の方針

番号は必ず `\label`/`\ref` で出し、本文に数字を直書きしない。**類別詞（図・表・式・節）を必ず付ける**（現状 `\ref{sec:experiment}と共通` は「4と共通」と読めて不自然）。

| 対象 | 確定表記 | 備考 |
|---|---|---|
| 図 | `図\ref{fig:xxx}` | |
| 表 | `表\ref{tab:xxx}` | |
| 式 | `式\eqref{eq:xxx}`（amsmath） | `\eqref` が括弧付き番号を自動生成 |
| 節・章 | `\ref{sec:xxx}節` | 番号は直書きしない |
| 文献 | `語~\cite{key}` | `~`（改行禁止）で語と密着。語の直後・句読点の前 |

label のプレフィックスを `fig:` `tab:` `eq:` `sec:` で統一する。

## 記号の方針（基本 LaTeX）

数式記号・演算子は**数式モード `$...$`** で書き、地の文に裸の Unicode を置かない。

| 用途 | 書き方 |
|---|---|
| 不等号 | `$>$` `$<$` `$\leq$` `$\geq$` |
| 掛け算 | `$\times$`（`×` を直書きしない） |
| ノルム・絶対値 | `$\|\mu\|$` `$|x|$` |
| マイナス | 数式中の `-`（`$-16.1$` など）。和文のハイフンと区別 |
| 引用符 | 和文は `「」`、欧文は LaTeX の `` `...' `` / ``` ``...'' ```（`“ ”` をそのまま打たない） |

## 機械的に検出する grep 例

```bash
# 句読点の混在（，．の文書に 、。 が残っていないか）
grep -rn "[、。]" section/ main.tex

# 全角コロン・全角英数の混入
grep -rn "：\|０-９\|Ａ-Ｚ" section/ main.tex

# 半角カタカナ
grep -rnP "[｡-ﾟ]" section/ main.tex

# 波ダッシュ/チルダの混在
grep -rn "~\|〜\|～" section/ main.tex
```

## 校正フロー

grep は「決まった字面」しか拾えない。送り仮名・漢字/かなの開き・想定外の表記ゆれは Agent で拾う、**2 段構え**にする。

### 段1: grep で機械検出

1. 上の grep で**機械検出できる違反**を一掃する（句読点・全角半角・カタカナ・ダッシュ）。
2. 用語・語彙レベルの統一は、その論文専用の用語辞書スキルがあれば併用する（例: [[clipscoreimprovement-proofread]]）。
3. 残りを #6〜#11 のチェックリストで通読点検する。

### 段2: Agent で表記ゆれを検出

grep のパターンで予測しきれない**表記レベルのゆれ**を Agent（例: `Explore`）に全文走査させる。観点:

- **送り仮名のゆれ**（行う/行なう、表す/表わす、など）。
- **漢字/かなの開き**の不統一（例えば/例えば、及び/および、更に/さらに）。
- **同義語の併用**（同じ概念を別語で呼ぶ：手法/方式、評価/検証 など）。
- **数値・単位・記号表記の不統一**で grep に書かなかったもの。

Agent への指示テンプレート:

```
section/*.tex と main.tex を全文読み、日本語の表記ゆれだけを報告して。
観点: 送り仮名、漢字/かなの開き、同義語の併用、数値・単位・記号の表記不統一。
論証や内容の良し悪しは見ない。各指摘は file:line・現状・推奨・理由の形で。修正はせず報告のみ。
```

4. 修正は**一括置換できるものだけ置換**し、文脈依存（同じ字面で意味が違う語）は1件ずつ確認する。
