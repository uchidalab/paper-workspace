# LaTeX 2カラム論文の体裁規則

2カラム構成の日本語論文（jsarticle + twocolumn）における図・表・要旨の配置規則。レイアウト崩れを防ぐための確定ルールを定める。

## 図・表の配置

### カラム内フロート（`figure` / `table`）

| 規則 | 確定 |
|---|---|
| 配置カラム | **右カラム**に置く。左カラムには置かない |
| 配置位置 | **カラム上部**に詰める。placement は `[t]` を基本とする |
| 参照との距離 | 参照箇所と同じページか隣接ページに配置する |

右カラムに配置するには、**フロート宣言をソース上で十分後ろ（左カラム相当の本文の後）に置く**。LaTeX は宣言位置以降の最初の適切なカラム上部に配置するため、左カラムを埋める本文の途中で宣言すると左に入ってしまう。

### 2カラムぶち抜きフロート（`figure*` / `table*`）

| 規則 | 確定 |
|---|---|
| 使用可否 | **横長の図・表に限り使用してよい** |
| 配置位置 | **ページ上部**。placement は `[t]` または `[!t]` |
| 宣言位置 | `figure*` は**次ページの上部**に配置される。参照箇所の近くに出すには、参照より前のページの本文中で宣言する |

### 配置の確認手順

1. `./scripts/build-latex.sh` でビルド
2. PDF を開き、以下を目視確認:
   - 単カラム図・表が左カラムに入っていないか
   - 図・表がカラム/ページの上部に詰まっているか
   - `figure*` / `table*` が参照箇所から離れすぎていないか
3. 違反があればフロート宣言の位置を調整して再ビルド

### よくある違反と対処

| 違反 | 原因 | 対処 |
|---|---|---|
| 図・表が左カラムに配置される | フロート宣言が左カラムの本文中にある | 宣言を右カラム相当の位置まで後ろに移動 |
| `figure*` が参照から遠いページに飛ぶ | 宣言が遅すぎて次ページに入らない | 宣言を1〜2段落前に移動 |
| 図・表がページ下部に配置される | `[b]` や `[h]` を指定している | `[t]` に変更 |

## 要旨・キーワード

| 規則 | 確定 |
|---|---|
| セクション番号 | **割り当てない**。`\section` や `\section*` を使わない |
| 配置 | `\twocolumn[...]` 内に `\noindent\textbf{要旨：}` の形式で記述する |
| 幅 | `minipage{\textwidth}` で全幅表示とする |

## grep 検出例

```bash
# placement が [t] 以外のフロート
grep -rn '\\begin{figure}\|\\begin{table}\|\\begin{figure\*}\|\\begin{table\*}' section/ | grep -v '\[t\]\|%'

# 要旨にセクション番号が付いていないか確認
grep -rn '\\section.*要旨\|\\section.*概要\|\\section.*Abstract' section/ main.tex
```

## 校正フロー

1. 上の grep で placement 指定を確認する
2. PDF をビルドして目視で図・表の配置カラムと位置を確認する
3. 違反があればフロート宣言の位置と placement を調整する
